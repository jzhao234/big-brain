## How it's organized

| Folder | What goes there |
| --- | --- |
| `inbox/` | Quick captures, unprocessed. Triage regularly. |
| `daily/` | One note per day: log, focus, quick tasks. |
| `projects/` | One note per project — goal, tasks, running log. The unit of "what am I working on". |
| `areas/` | Ongoing responsibilities with no end date (health, team, finances). |
| `notes/` | Evergreen knowledge notes, linked with `[[wikilinks]]`. |
| `people/` | One note per person you work or deal with. |
| `reference/` | External facts: docs, decisions, account details, how-tos. |
| `archive/` | Anything no longer active. Nothing is deleted, only archived. |
| `templates/` | Note templates used by big-brain and Obsidian. |

## Ground rules

1. **Capture first, organize later.** Anything worth keeping goes to `inbox/` immediately.
2. **Link generously.** A note without `[[links]]` is a note you'll never find again.
3. **Projects drive tasks.** Every task lives in a project or a daily note — nowhere else.
4. **The log is the memory.** Decisions and progress get one line in a project Log or the daily note, at the moment they happen.
5. **Review weekly.** Run the `weekly-review` prompt: triage inbox, prune projects, reschedule tasks.
